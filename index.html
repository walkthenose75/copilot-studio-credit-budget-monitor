One currency runs every Microsoft AI agent

Before you compare plans or argue about budget, there's one idea the whole story hangs on: the Copilot Credit. Every agent response, every automated action, every AI-powered task across Copilot Studio, Power Automate, Dynamics 365 agents, and Copilot Chat is paid for in this single unit. Understand the credit, and everything that follows — rates, plans, governance — is just spending it or buying it.

Four supply routes for the same credit

You spend one currency — but you can source it four ways, from zero-commitment metering to deep annual discounts. They're not mutually exclusive; most customers layer a prepaid plan on top of pay-as-you-go.

Pay-as-you-go

Annual

Annual · broad

How to purchase

Three purchase paths, two portals. Pay-as-you-go is set up against an Azure subscription and bills monthly in arrears at the list rate, doubling as the overage safety net. Copilot Credit packs are bought as a subscription add-on in the Microsoft admin center — the simplest route for steady, predictable monthly usage. The P3 pre-purchase plan is a one-time, upfront Azure reservation for a discounted annual commitment.

② Pay-as-you-go meter — the pipe

Azure billing policy (Azure subscription)

The metering layer every credit flows through at retail USD. The prepaid plan requires this billing policy — and any overage beyond your prepaid pool simply bills here.

⚠️

The one stand-alone exception: a monthly capacity pack can run without an Azure subscription — but then there's no overage path, so service pauses when the pack empties. P3 / ACU reservations always ride on the pay-as-you-go layer above.

Advanced · which benefit pays first when you own several (precedence)

Most customers buy one plan, so this rarely matters. It only applies if you hold multiple benefits at once — then they drain in this order before pay-as-you-go:

  1. Foundry PTU Reservation — pays provisioned-throughput usage first
  2. Fabric Capacity Reservation — pays Fabric capacity usage
  3. Copilot Credit P3 (CCCU) — Copilot-specific prepaid pool
  4. Microsoft Agent P3 (ACU) — broad prepaid pool across platforms
  5. Pay-as-you-go — anything beyond your benefits, at retail

Reservations always apply before prepurchase plans. If you own only one, it simply pays — there's no cascade.

What the Agent (ACU) plan covers across Microsoft

ACU is the broad prepaid pool: one commitment that pays down AI usage across four Microsoft platforms. The biggest surface is Microsoft Foundry — 34 covered services.

What are ACUs — and what can they be used for? i

An Agent Commit Unit (ACU) is a prepaid, discounted commit unit purchased through Azure Reservations. 1 ACU = $1 = 100 Copilot Credits. Purchased ACUs automatically pay down qualifying usage in US dollars from a single shared pool — no redeployment or reassignment needed — for a one-year term. Unlike the Copilot Credit P3 plan (which covers Copilot Credit usage across Copilot Studio, D365 first-party agents, and Copilot), ACUs span multiple Microsoft AI services, making them the unified budget for organizations running agents across more than one platform.

Eligible usage the pool can pay for i

  • Microsoft Copilot Studio — Copilot Credits for custom agents, Dynamics 365 first-party agents, and Copilot
  • Microsoft Foundry — PTU (Provisioned Throughput Unit) hours for AI model deployments
  • Microsoft Fabric — Copilot capacity, metered on tokens processed
  • GitHub — GitHub Copilot and other GitHub AI usage (via linked Azure subscription)

How the plan works

  • Buy a tier of ACUs in the Azure portal › Reservations blade
  • Eligible usage deducts automatically from the pool in USD
  • More ACUs purchased = larger discount (5% / 10% / 15%)
  • One-year term, auto-renews by default; unused ACUs do not roll over
  • Requires Owner or Reservation Purchaser role (or EA / CSP equivalent)
  • Plan discount is applied independently and does not stack with other discounts

Sizing example (Microsoft): 1,500,000 Copilot Credits ($15,000 at PayGo) + 5,000 Foundry PTU hours ($5,000) = $20,000 pay-as-you-go. A Tier 1 plan of 20,000 ACUs costs $19,000 — about a 5% / $1,000 saving for predictable workloads.

Source: Microsoft Agent Prepurchase Plan (Microsoft Learn).

Set up & allocate — packs, pay-as-you-go, and P3

Buying credits is only half the job. Each funding path has its own setup. Credit packs allocate tenant capacity to environments; pay-as-you-go links an Azure subscription as the metered safety net; and P3 links the subscription, then allocates the prepurchased pool. The detailed link and allocate flows live under P3 below — packs and pay-as-you-go reuse them.

Credit packs · Microsoft 365 admin center

Allocate pack capacity to environments i

Packs need no Azure subscription. Once purchased, the prepaid Copilot credit capacity is pooled at the tenant and you put it to work by allocating it to each environment using capacity management (Manage Capacity) in the Power Platform admin center — the same allocation flow shown under P3 below.

  • Capacity is enforced monthly; unused credits do not roll over
  • Reallocate unconsumed credits between environments at any time
  • If a pack empties with no pay-as-you-go linked, agent runs pause until the next month

Source: Manage Copilot Studio credits and capacity (Microsoft Learn).

Pay-as-you-go · Azure subscription

Link an Azure subscription as the metered safety net i

Pay-as-you-go meters every credit at retail and bills monthly in arrears to a linked Azure subscription. It is also the overage safety net: when a pack or P3 pool runs dry, usage flows here instead of stopping. Setup is to link the Azure subscription to your environments via a billing plan in the Power Platform admin center — the exact billing-plan flow shown under P3 below.

Reference: Set up a pay-as-you-go plan (Microsoft Learn).

Overage enforcement is the safety net

When an environment runs past its allocation, Microsoft allows a grace band, then enforces. You choose how to respond — reallocate, buy more, or enable pay-as-you-go so nothing ever hard-stops.

When capacity is exceededWhat happens
Reallocate capacityMove unused credits from other environments in the tenant.
Purchase more creditsTop up and assign to the affected environment.
Enable pay-as-you-goLink an Azure billing policy — overage bills through and enforcement no longer applies.
Do nothing (125%)Past the grace band, custom agents are disabled until capacity is restored.

Enforcement targets custom-agent and flow execution; classic and generative answers continue. Pay-as-you-go is the recommended business-continuity safety net.
