Executive overview

Three ways to buy Copilot Credits, one universal currency

Pick the model that matches the usage pattern

Credit capacity packs

Steady, predictable usage

MCS license subscription. Buy the number of packs you expect to consume each month. Requires a tenant license in the M365 Admin Center. Monthly expiry.

Pay-as-you-go

No prior usage history

Postpay for actual consumption through an Azure subscription billing policy. No upfront commitment. Ideal safety net so critical agents never stop.

Pre-Purchase Plan (P3)

Larger or variable demand

One-year pre-paid CCCU pool. Built-in tiered discounts up to 20% off PayGo, annual (not monthly) expiry, MACC eligible, tracked in the Azure Portal. No M365 Admin Center license needed.

Overage and enforcement, step by step i

  • Under 100%: normal consumption from allocated capacity.
  • 100 to 125%: grace overage allowed so business processes are not blocked.
  • At 125%: custom agents disabled; admin emailed and notified in PPAC. Users see "This agent is currently unavailable. It has reached its usage limit."

Avoid enforcement: reallocate capacity, purchase more credits, or link a pay-as-you-go meter so overage bills to Azure instead of blocking agents.

Agent flow enforcement is different

When prepaid capacity is fully consumed, new agent flow runs are blocked, but the parent agent continues to function for classic answers, generative answers, and agent actions. In-progress runs finish. M365 Copilot licensed users and test runs are exempt. Resolve by reallocating capacity, buying more credits, or enabling PayGo.
